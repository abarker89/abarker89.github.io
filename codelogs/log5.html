Images

This page explains how images are used in HTML and CSS, how they differ, and which image formats are best for different situations.

Using Images in HTML vs CSS

1. Using the <img> Tag in HTML

  • Uses the <img> element directly in the HTML.
  • Displays an image as part of the page content.
  • Supports alt text for accessibility.
  • Has intrinsic dimensions and can affect layout flow.
<img src="photo.jpg" alt="Sunset over mountains" width="400" height="300">

2. Using background-image in CSS

  • Uses the CSS property background-image on an element.
  • Displays an image as decoration or background, not part of the main content.
  • Does not support alt text or affect document structure.
  • Good for patterns, banners, and visual design elements.
div {
  background-image: url("photo.jpg");
  background-size: cover;
  background-position: center;
}

Key Differences

  • HTML <img>: For meaningful, content-related images.
  • CSS background-image: For decorative or layout purposes.
  • Accessibility: <img> supports alt; background images do not.
  • Layout: <img> takes up space in the flow; background images fill an elementâ€™s box.

Other CSS Properties Affecting Background Images

  • background-size: Controls how the image is scaled (e.g. cover or contain).
  • background-position: Sets where the image is placed within the element (e.g. center, top left).
  • background-repeat: Determines if the image repeats (e.g. no-repeat, repeat-x).

Dimensions vs Resolution

Dimensions are the width and height in pixels, while resolution refers to how many pixels fit into a given physical space (like pixels per inch).

Extra Credit: Image File Formats

GIF

  • Supports simple animation and transparency.
  • Limited to 256 colors, so not ideal for photos.
  • Best for small animations or icons.

PNG

  • Lossless compression (no quality loss).
  • Supports full transparency.
  • Great for logos, icons, and images with text.

JPG/JPEG

  • Lossy compression (some quality loss for smaller file size).
  • Best for photos and detailed images.
  • Does not support transparency.

WebP

  • Modern format that supports both lossy and lossless compression.
  • Supports transparency and animation.
  • Smaller file sizes than PNG or JPG at similar quality.

SVG

  • Vector-based, not pixel-based.
  • Scales infinitely without losing quality.
  • Great for logos, icons, and simple illustrations.

Summary of When to Use Each

  • GIF: Simple animations.
  • PNG: Crisp, transparent graphics.
  • JPEG: Photographs and detailed images.
  • WebP: Modern choice for web performance.
  • SVG: Scalable graphics and icons.
